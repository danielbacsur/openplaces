Every result yielded by [`places.search`](/openplaces/openplaces/places/search) and [`places.stream`](/openplaces/openplaces/places/stream) is a `Place`: a single flat object with 27 top-level fields. Everything OpenPlaces can parse out of a result is surfaced on the object directly, rather than nested behind sub-resources.

`Place` is a Zod schema exported from the package root, and the TypeScript type is inferred from it. You can use the schema to validate data that crosses a process or serialization boundary:

```ts
import { Place } from "openplaces";

const place = Place.parse(data); // throws if the shape is invalid
const checked = Place.safeParse(data); // { success, data } | { success, error }
```

Internally, every candidate parsed from a results page is passed through `Place.safeParse`, and candidates that fail validation are dropped — so anything the library yields already conforms to this schema. Fields that could not be parsed are absent, not `null`.

Only `id`, `name`, `latitude`, and `longitude` are always present; every other field is optional.

## Nested fields

Two fields carry structure; everything else is a scalar or a string array.

`hours` describes the opening state at the time of the search, split into three optional strings:

- `status` — the short state, e.g. `"Open"` or `"Closed"`.
- `detail` — the remainder of the status line, e.g. `"Closes 10 pm"`.
- `color` — a hex color derived from the color Google renders the status in, e.g. green while open.

`services` is an array of `{ label: string; available: boolean }` entries taken from the service options of the listing — for a restaurant, typically entries like `{ label: "Dine-in", available: true }` or `{ label: "Delivery", available: false }`. For sponsored results the entries come from the ad card's service badges instead.

Two more fields deserve a note. `accessible` is derived from the wheelchair-accessibility attribute of the listing, so it is `true`, `false`, or absent when the listing does not state it. `sponsored` is only ever `true` or absent — organic results omit the field rather than carrying `false`.
